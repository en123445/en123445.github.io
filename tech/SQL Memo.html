SQL Memo

知识点​

  • 聚合函数和 GROUP BY 子句
  1. 只能写在 SELECT 子句之中
  2. GROUP BY 子句中不能使用 SELECT 子句中列的别名
  3. GROUP BY 子句的聚合结果是无序的
  4. WHERE 子句中不能使用聚合函数

  • SELECT 语句的执行顺序 FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY → LIMIT

  • DISTINCT 与 GROUP BY的使用时机 "想要删除选择结果中的重复记录"时 使用 DISTINCT ， "想要计算汇总结果"时 使用 GROUP BY

  • WHERE 与 HAVING WHERE 子句 = 指定行所对应的条件 （速度快） //WHERE 子句中不能使用聚合函数!! HAVING 子句 = 指定组所对应的条件 ↑↑↑↑↑↑ 优先选WHERE，再选HAVING（用WHERE过滤掉不要的数据，减少数据量）↑↑↑↑↑↑

子句的书写顺序

1. SELECT 子句 → 2. FROM 子句 → 3. WHERE 子句 → 4. GROUP BY 子句 → 5. HAVING 子句 → 6. ORDER BY 子句 → 7.LIMIT
  • 【优化原则】 小表驱动大表：小的数据集驱动大的数据集
SELECT * from A WHERE id in (select id from B) -- 当B的数据集小于A时，in优于EXISTS
等价于：
for SELECT id FROM B
for SELECT * FROM A WHERE A.id = B.id
SELECT * from A WHERE EXISTS in (select * from B WHERE B.id = A.id)  -- 当A的数据集小于B时，EXISTS优于in
等价于：
for SELECT * FROM A
for SELECT * FROM B WHERE B.id = A.id

EXISTS的思考逻辑（习惯"肯定 <=> 双重否定"之间的转换） 所有的 x 都满足条件 P = 不存在不满足条件 P 的 x

  • EXISTS（存在） 先运行主查询一次 再去子查询里查询与其对应的结果 如果存在，返回ture则输出,反之返回false则不输出,再根据主查询中的每一行去子查询里去查询.

  • 外部查询 与 子查询 执行顺序如下：

  1. 首先执行一次外部查询
  2. 对于外部查询中的每一行分别执行一次子查询，而且每次执行子查询时都会引用外部查询中当前行的值。
  3. 使用子查询的结果来确定外部查询的结果集。 如果外部查询返回100行，SQL 就将执行101次查询，一次执行外部查询，然后为外部查询返回的每一行执行一次子查询。

  • IN（包含） in()后面的子查询是返回结果集的,换句话说执行次序和exists()不一样. 子查询先产生结果集,然后主查询再去结果集里去找符合要求的字段列表去.符合要求的输出,反之则不输出.

  • 执行sql脚本命令 mysql> source xxx.sql (文件绝对路径)

-- 标量子查询,必须而且只能返回 1 行 1列的结果,因此标量子查询的返回值可以用在 = 或者 <> 这样需要单一值的比较运算符之中

-- 关联子查询
-- 对于外部查询返回的每一行数据，内部查询都要执行一次。
-- 先执行外层查询，中间结果的第n条记录（的匹配条件）传递给子查询，子查询结果返给外层查询继续做比较，得到最终结果
/* 例:
SELECT P1 结果的第一条记录的 P1.product_type 传给子查询
用P1.product_type = P2.product_type的关联条件得到子查询的AVG()结果
子查询结果返回外层，继续做 > 比较，得到P1.product_type条件的查询结果
外层查询的每一条结果循环传入子查询，重复上述比较过程
※product_type相同的行数据重复计算时由sql优化效率
*/
SELECT product_type , product_name, sale_price
FROM Product AS P1
WHERE sale_price > (SELECT AVG(sale_price)
    FROM Product AS P2
    WHERE P1.product_type = P2.product_type
    GROUP BY product_type);

/*
两个集合的结构要一致,对应的字段数,字段类型
将两个集合用 UNION ALL 关键字合并,这里的结果是有重复的所有集
将上面的所有集 GROUP BY id
最后 HAVING COUNT(id)=1,等于1的意思是只出现了一次,所以这个是差集,如果等于2,那么就是交集
*/

-- 交集 （无函数，代替 INTERSECT）
SELECT P.product_id, P.product_name
FROM (
    SELECT product_id, product_name
    FROM Product
    UNION ALL
    SELECT product_id, product_name
    FROM Product2
    ) AS P -- 派生表需要用别名识别
GROUP BY P.product_id HAVING COUNT(P.product_id) = 2; 

-- 差集 （无函数）
-- 只在一个集合中出现
SELECT P.product_id, P.product_name
FROM (
    SELECT product_id, product_name
    FROM Product
    UNION ALL
    SELECT product_id, product_name
    FROM Product2
    ) AS P -- 派生表需要用别名识别
GROUP BY P.product_id HAVING COUNT(P.product_id) = 1; 

-- 记录减法 （无函数，代替EXCEPT）
-- 只在左边的集合出现
SELECT product_id, product_name
FROM Product
WHERE product_id NOT IN (SELECT product_id FROM Product2)

或者
SELECT P1.product_id, P1.product_name
FROM Product AS P1 LEFT JOIN Product2 AS P2

一对一或一对多关系的两个集合，在进行连接操作后行数不会（异常地）增加

-- 联结
-- INNER JOIN （内联结）（同时存在于两张表的数据）（两表无主次关系，匹配上的才有效）
SELECT SP.shop_id, SP.shop_name, SP.product_id, P.product_name, P.sale_price
FROM ShopProduct AS SP INNER JOIN Product AS P
ON SP.product_id = P.product_id
WHERE SP.shop_id = '000A';

-- OUTER JOIN （外联结）（只要存在于其中一张表）（有一个主表）
SELECT SP.shop_id, SP.shop_name, SP.product_id, P.product_name, P.sale_price
FROM ShopProduct AS SP RIGHT OUTER JOIN Product AS P --  LEFT 和 RIGHT 指定谁是主表，结果中会包含主表内所有的数据
ON SP.product_id = P.product_id;

-- 模拟全外连接
SELECT P1.product_id, P1.product_name, P2.product_id, P2.product_name FROM Product AS P1
LEFT JOIN Product2 AS P2 ON P1.product_id = P2.product_id
UNION
SELECT P1.product_id, P1.product_name, P2.product_id, P2.product_name FROM Product AS P1
RIGHT JOIN Product2 AS P2 ON P1.product_id = P2.product_id
